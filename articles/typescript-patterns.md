# TypeScript 日常开发中的实用模式

使用 TypeScript 的第四年，我已经很难想象没有类型的日子。但随着经验增长，我发现用好 TypeScript 不仅仅是写类型注解，更在于掌握一些关键的模式和思维习惯。

## 辨析联合类型

辨析联合类型（Discriminated Unions）是我最常用的模式之一。当一个值可能有多种形态时，用一个共同的字面量字段（比如 `type` 或 `status`）来区分它们，TypeScript 就能在每一个分支中精确推断出对应的类型。这让处理复杂的状态机、API 响应和表单数据变得安全而优雅。

## 模板字面量类型

模板字面量类型（Template Literal Types）是 TypeScript 4.1 引入的特性，但我直到最近才开始充分利用它们。你可以用它们来定义精确的字符串格式——比如事件名称、路由路径、CSS 单位——并在编译时就捕获拼写错误。

## 类型守卫

类型守卫（Type Guards）和类型断言函数让类型收窄变得显式和可复用。与其在每个 `if` 分支中依赖自动的类型推断，不如写一个明确的守卫函数，既提高了可读性，也减少了出错的可能。

## 不要过度使用泛型

最后想说的一点是：不要过度使用泛型。泛型是强大的工具，但如果你发现自己在写三层嵌套的泛型约束，可能应该重新审视一下设计。好的类型系统应该让代码更清晰，而不是更费解。
