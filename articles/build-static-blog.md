# 构建极简静态博客的思考与实践

过去几年，我试过不少博客方案：WordPress、Hexo、Hugo、Notion + 第三方渲染服务。它们各有优势，但总有一些细节让我不满意——要么太臃肿，要么定制起来很痛苦，要么加载速度不够理想。

于是我决定自己动手，从零写一个静态博客。核心原则只有三个：**极简、快速、零依赖**。整个项目只有三个文件——一个 HTML、一个 CSS、一个 JavaScript。不需要构建工具，不需要 `npm install`，不需要任何框架。

技术选型上，我选择了纯 HTML + CSS + JavaScript。CSS 自定义属性（Custom Properties）处理深色/浅色主题切换，URL hash 做前端路由，localStorage 记住用户偏好。没有使用任何第三方库。

文章数据直接写在一个 JavaScript 对象里。新增文章就是在数组里加一条记录，修改就是编辑对应条目。不需要 CMS 后台，不需要数据库，不需要 API。对于一个个人博客来说，这已经足够了。

CSS 自定义属性是实现双主题的关键。我在 `:root` 中定义深色主题的变量，在 `[data-theme='light']` 中覆盖为浅色主题的变量。切换主题只需要改变 `html` 元素上的 `data-theme` 属性，所有颜色会自动响应。配合 CSS transition，切换过程非常平滑。

前端路由使用 `window.location.hash`。通过监听 `hashchange` 事件，在同一个 HTML 文件中实现了首页、分类页、文章详情页三种视图的切换。支持浏览器的前进和后退按钮，也支持直接通过 URL 访问特定页面。

整个博客的核心代码不到 300 行。加载速度极快，没有任何网络请求（除了字体回退），对 SEO 友好，部署到 GitHub Pages 或 Vercel 上只需几分钟。如果你也在考虑搭建个人博客，不妨试试从零写一个——你会学到很多。
