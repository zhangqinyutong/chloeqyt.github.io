# 深入理解 CSS 自定义属性

很多人第一次接触 CSS 自定义属性时，会觉得它「不就是 Sass 变量吗」。这个类比可以帮助入门，但远远低估了自定义属性的能力。

最关键的区别在于：**CSS 自定义属性工作在级联（cascade）层面，而不是编译时**。这意味着它们是动态的、可继承的、可以被媒体查询和选择器覆盖的。Sass 变量在编译后就消失了，而自定义属性在浏览器中实时生效。

举一个典型的应用场景：深色模式。使用 CSS 自定义属性，你可以在 `:root` 中定义一套颜色变量，然后在 `[data-theme='dark']` 中覆盖它们。切换主题只需要一行 JavaScript 修改 `data-theme` 属性，所有使用了这些变量的元素会自动更新颜色。用预处理器变量做不到这一点。

另一个强大特性是继承。自定义属性可以被子元素继承，你可以在组件的最外层设置一个值，内部所有元素都能访问到。这对于构建可组合的 UI 组件非常有用。

配合 JavaScript 的 `setProperty` 和 `getPropertyValue` 方法，你甚至可以在运行时动态读写自定义属性。这为交互式 UI 提供了很多可能性，比如根据用户输入实时调整间距、颜色或字体大小。

当然，自定义属性也有局限——IE 不支持（但这已经不重要了），以及不能用于媒体查询的条件中。但总体来说，它们是现代 CSS 中最重要的特性之一，值得深入掌握。
