# 《程序员的修炼之道》—— 二十年后再读

Andrew Hunt 和 David Thomas 的《程序员的修炼之道》是那种每隔几年就该重读一次的书。今年我读了二十周年纪念版，发现它新增了不少关于并发、函数式编程和数字化转型的内容，但最打动我的，还是那些二十年前就写下的核心原则。

## 正交性

「正交性」是我在日常工作中引用最多的概念。两个组件如果互不影响，它们就是正交的。正交的系统更容易修改、测试和理解。每当我设计 API 或拆分模块时，我都会问自己：这个改动会波及多少地方？如果答案超过一，我就会重新审视设计。

## 曳光弹

「曳光弹」这个概念同样深刻。相比试图一次性构建完整的系统，不如先实现一条端到端的「子弹路径」——它不需要完整的功能，但能验证所有组件是否正确连接。这和现代敏捷开发中的「垂直切片」是一个道理。

## 知识组合

书中的「知识组合」比喻也令我印象深刻。程序员的真正资产不是代码，而是知识和经验。像管理金融投资组合一样管理你的知识——分散风险、定期评估、持续投入。在技术日新月异的今天，这个建议比以往任何时候都重要。

## 好习惯的复利

二十周年纪念版还强调了「好软件的秘诀在于好习惯」。工具可以换，语言会过时，但编写可维护代码的习惯、测试的纪律、持续学习的态度——这些才是程序员真正的护城河。
